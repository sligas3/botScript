# Reglas para Guía de Inicio Rápido (README.md)

## Estructura Obligatoria en README.md
- **¿Qué hace?**: Una línea explicando el propósito del servicio
- **¿Para quién?**: Audiencia objetivo en 1-2 líneas
- **Prerequisitos**: Lista mínima de dependencias
- **Instalación**: Comandos copy-paste para ejecutar
- **Primer uso**: Ejemplo funcional en < 5 minutos
- **Enlaces**: Referencias a /docs/ para información detallada

## Contenido del Quick Start
### Descripción Ultra-Concisa
- Máximo 2 párrafos explicando qué resuelve el servicio
- Usar analogías simples para conceptos complejos
- Evitar jerga técnica innecesaria

### Instalación Express
- Un solo comando para instalar/ejecutar cuando sea posible
- Docker compose o scripts automatizados
- Variables de entorno mínimas requeridas

### Ejemplo "Hello World"
- Request/response más simple posible
- Datos de prueba incluidos
- Resultado esperado claramente definido

## Reglas de Escritura
- Usar voz activa: "El servicio procesa" vs "Los datos son procesados"
- Verbos de acción: crear, enviar, procesar, validar
- Evitar palabras como "simplemente", "obviamente", "fácilmente"
- Incluir tiempo estimado para cada paso

## Validación
- Cada ejemplo debe ser ejecutable sin modificaciones
- Incluir comando para verificar que funciona
- Proporcionar troubleshooting para errores comunes
