# Reglas Core para Documentación

## Principios de Documentación
- **Documentación como código**: Versionada junto al código fuente
- **Principio DRY**: Evitar duplicación de información
- **Documentación viva**: Actualizada automáticamente cuando sea posible
- **Accesibilidad**: Clara para diferentes niveles técnicos

## Estilo y Formato
- Usar markdown para documentación técnica
- Títulos descriptivos y jerarquía clara (H1-H6)
- Párrafos cortos y listas para mejorar legibilidad
- Código inline con backticks, bloques con triple backticks
- Usar tablas para comparaciones y especificaciones

## Contenido Técnico
- Incluir ejemplos ejecutables siempre que sea posible
- Proporcionar contexto antes de detalles técnicos
- Explicar el "por qué" además del "cómo"
- Usar diagramas para conceptos complejos
- Incluir troubleshooting para problemas comunes
